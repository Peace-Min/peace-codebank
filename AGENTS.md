# Peace.Codebank

## 프로젝트 개요

`Peace.Codebank`는 개인용 C# 재사용 코드 저장소다.

이 저장소의 목적은 공용 프레임워크나 DLL 패키지를 만드는 것이 아니다. 여러 프로젝트에서 다시 사용할 만한 로직을 모아 두고, `net472`와 `net8.0`에서 검증한 뒤 필요한 파일만 복사해서 가져다 쓰는 것을 목표로 한다.

## 방향성

- 모듈은 작고 명확해야 한다.
- 모듈은 가능한 한 복사만으로 사용할 수 있어야 한다.
- 특정 앱의 UI, 인프라, 실행 환경에 강하게 묶인 코드는 넣지 않는다.
- 장기적으로 다시 쓸 가능성이 높은 로직만 보관한다.
- 모든 모듈은 테스트 프로젝트를 통해 계속 검증한다.

## 글로벌 규칙

1. 모든 재사용 모듈은 `net472`와 `net8.0`을 모두 지원해야 한다.
   - `net8.0`에서만 사용할 수 있는 문법, API, 타입, 편의 메서드는 사용하지 않는다.
   - 프레임워크 차이가 있는 경우 가장 보수적인 공통 기준으로 작성한다.

2. 외부 코드는 보안과 라이선스를 검토한 뒤 이해하고 정리해서 반영한다.
   - 외부 코드를 그대로 복사하지 않는다.
   - 저장소 기준에 맞게 구조, 이름, 테스트를 정리한 뒤 반영한다.
   - 라이선스, 출처, 보안 검토를 우회하려는 방식으로 작업하지 않는다.

3. C# 코드는 저장소 코딩 규칙을 반드시 따른다.
   - `try-catch`를 사용할 때는 의미가 명확한 예외 형식만 잡는다.
   - 일반 `catch` 또는 광범위한 `catch (Exception)`은 특별한 사유가 없는 한 사용하지 않는다.

4. 주석은 꼭 필요한 경우에만 작성한다.
   - 주석은 짧고 명확하게 작성한다.
   - 주석 문장은 `.`으로 끝낸다.

5. 모든 `README.md`는 한국어로 작성한다.
   - 루트 `README.md`와 각 모듈 `README.md` 모두 한국어를 사용한다.

## C# 코딩 규칙

### 기본 스타일

- 파일당 타입 하나를 원칙으로 한다.
- 파일 이름은 대표 타입 이름과 동일하게 맞춘다.
- 네임스페이스는 파일 범위 네임스페이스를 우선한다.
- `using`은 네임스페이스 바깥에 두고 `System` 계열을 먼저 정렬한다.
- 접근 제한자는 생략하지 않는다.
- private 인스턴스 필드는 `_camelCase`를 사용한다.
- 타입, 속성, 메서드, 상수는 `PascalCase`를 사용한다.
- 매개변수와 지역 변수는 `camelCase`를 사용한다.
- 지역 변수는 초기값이 있으면 `var`를 사용한다.
- 초기값이 없어 `var`를 사용할 수 없는 경우에만 명시형을 사용한다.
- `ArgumentNullException.ThrowIfNull`처럼 런타임 의존성이 생기는 편의 API 대신 프레임워크 공통 방식의 명시적 가드를 사용한다.

### 타입 내부 선언 순서

같은 종류의 멤버 안에서는 `public -> internal -> protected -> private` 순서를 따른다.

타입 내부의 기본 순서는 다음과 같이 맞춘다.

1. 상수.
2. static readonly 필드.
3. 인스턴스 필드.
4. 생성자.
5. 속성.
6. 메서드.
7. 중첩 타입.

### 예외 처리 규칙

- 예외를 잡는 이유가 분명할 때만 `try-catch`를 사용한다.
- `catch`에서는 처리 가능한 예외 형식만 명시적으로 선언한다.
- 예외를 삼키지 않는다.
- 로그만 남기고 계속 진행해야 할 때도 왜 안전한지 코드에서 드러나야 한다.

### 주석 규칙

- 코드만으로 의도가 드러나면 주석을 쓰지 않는다.
- 알고리즘의 제약, 외부 규격 대응, 프레임워크 차이처럼 코드만으로 설명이 부족한 경우에만 주석을 쓴다.
- 주석은 문장형으로 작성하고 `.`으로 끝낸다.

## 작업 지침

- 추상화보다 복사와 재사용 편의성을 우선한다.
- 모듈 하나를 복사했을 때 필요한 의존성도 함께 파악할 수 있어야 한다.
- 모듈을 추가하거나 수정하면 테스트도 함께 추가하거나 갱신한다.
- 사용법이나 제약이 있으면 해당 모듈의 `README.md`에 반드시 적는다.
- 저장소 규칙은 `AGENTS.md`, 코드 스타일은 `.editorconfig`, 커밋 규칙은 `docs/COMMIT_CONVENTION.md`를 기준으로 유지한다.

## 저장소 구조

- `Codebank/Modules/<Category>/<ModuleName>/src`: 재사용할 소스 파일.
- `Codebank/Modules/<Category>/<ModuleName>/tests`: 모듈 테스트 코드.
- `Codebank/Modules/<Category>/<ModuleName>/README.md`: 모듈 설명 문서.
- `TestApp/Peace.Codebank.Tests`: 저장소 검증 프로젝트.

## 검증

- `dotnet build .\Peace.Codebank.sln`.
- `dotnet test .\Peace.Codebank.sln`.
