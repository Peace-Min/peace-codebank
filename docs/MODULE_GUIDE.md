# 모듈 가이드

## 목적

각 모듈은 다른 코드베이스로 복사할 때 추가 정리가 거의 필요 없도록 유지한다.

## 기본 구조

```text
Codebank/Modules/<category>/<module-name>/
  README.md
  src/
  tests/
```

## 규칙

- 모듈은 가능한 한 자체 완결적이어야 한다.
- 다른 모듈 의존성은 복사 편의성을 해치지 않는 범위에서만 허용한다.
- 앱 전용 설정, UI 로직, 특정 인프라 가정은 넣지 않는다.
- POCO와 작은 인터페이스를 우선한다.
- 네임스페이스는 안정적으로 유지하고 모듈 범위를 드러내야 한다.
- 필요한 NuGet 패키지는 모듈 `README.md`에 문서화한다.

## 새 모듈 추가 절차

1. 모듈 폴더를 만들고 `src`, `tests`, `README.md`를 추가한다.
2. 재사용할 파일만 `src`에 둔다.
3. 검증 코드를 `tests`에 추가한다.
4. mock이 필요하면 인터페이스에 의존하고 테스트에서 동작을 검증한다.
5. 외부 패키지가 필요하면 `Directory.Packages.props`와 테스트 프로젝트를 함께 갱신한다.

## 복사 체크리스트

- 모듈 `src` 폴더의 파일을 모두 복사한다.
- 모듈 `README.md`를 확인한다.
- 문서에 적힌 의존성 등록이나 패키지 참조를 함께 반영한다.
- 대상 소프트웨어에 맞추고 싶을 때만 네임스페이스를 변경한다.

## 확장 기준

모듈이 가볍고 독립적이라면 현재의 단일 테스트 프로젝트로 충분하다.
특정 모듈이 무거운 의존성이나 충돌 가능성을 가져오기 시작하면 메인 테스트 프로젝트를 오염시키지 말고 전용 테스트 프로젝트를 분리한다.
